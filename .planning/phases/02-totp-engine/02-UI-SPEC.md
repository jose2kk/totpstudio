## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps between inline action buttons (`gap-1`), input adornment internal padding (`gap-0.5`) |
| sm | 8px | Compact element spacing — error text margin-top (`mt-2`), toggle group item padding |
| md | 16px | Default section spacing — label to control gap, card content padding |
| lg | 24px | Between major sections: secret input → parameters row → code display |
| xl | 32px | Card vertical padding |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Touch targets for icon buttons inside the secret field: minimum 28px (`size-7`) per shadcn `size="icon-sm"` — verified in button.tsx
- Input right-padding for 3 icon buttons: `pr-24` (96px) to prevent text overlap with adornments

## Color

All tokens are oklch CSS variables from `src/app/globals.css`. Light and dark values listed.

| Role | Light Value | Dark Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) | oklch(0.145 0 0) | Page background |
| Secondary (30%) | `--card` oklch(1 0 0) / `--muted` oklch(0.97 0 0) | oklch(0.205 0 0) / oklch(0.269 0 0) | Card surface, segmented control track background |
| Accent (10%) | `--primary` oklch(0.205 0 0) | oklch(0.922 0 0) | See reserved list below |
| Destructive | `--destructive` oklch(0.577 0.245 27.325) | oklch(0.704 0.191 22.216) | Invalid input border + error text |

Accent (`--primary`) reserved for:
- Active/selected state of ToggleGroup items (algorithm, digits, period segmented controls)
- Copy confirmation flash (brief `bg-primary` on copy icon button after successful copy)

Countdown progress bar colors (semantic, outside shadcn tokens — use Tailwind utility classes):
- >66% remaining: `bg-green-500` (green)
- 33–66% remaining: `bg-yellow-500` (yellow/amber)
- <33% remaining: `bg-red-500` (red)

These must be legible in both light and dark themes. `bg-green-500` / `bg-yellow-500` / `bg-red-500` pass against both `--background` light (white) and dark (near-black) at the progress bar's 4px height. Source: CONTEXT.md D-10, RESEARCH.md Pattern 3.

Invalid state colors:
- Input border: `border-destructive` (`--destructive`)
- Focus ring: `ring-destructive/20`
- Error text: `text-destructive text-sm`

Paused/invalid countdown bar: `bg-muted-foreground` / `opacity-50` — grey, no animation.

## Layout Contract

Single card layout. All TOTP UI lives inside the existing `Card` component in `src/app/page.tsx`.

Vertical stacking order (top to bottom):
1. Secret input row (with inline eye/copy/generate icon buttons)
2. Inline error text (conditionally visible below secret input)
3. Parameters row — three labeled segmented control groups side by side
4. TOTP code display (large monospace, centered, with copy button)
5. Countdown progress bar with right-aligned seconds label

Container: `max-w-3xl mx-auto px-4` (established in Phase 1 — do not change).

## Interaction States

### Secret Input Field

| State | Visual |
|-------|--------|
| Empty (initial) | No border highlight, no error. Placeholder: "Enter base32 secret" in muted color. |
| Valid + masked | Default border, monospace dots (password type). |
| Valid + revealed | Default border, monospace characters. |
| Invalid (non-empty) | `border-destructive` ring, error text below: "Invalid base32 secret" |
| Focused | Default `ring-ring/50` focus ring (existing shadcn convention). |

### Icon Buttons (inside secret field)

| Button | Icon | Action | Tooltip / aria-label |
|--------|------|--------|----------------------|
| Toggle visibility | `Eye` / `EyeOff` | Toggles `type="password"` ↔ `type="text"` | "Show secret" / "Hide secret" |
| Copy secret | `Clipboard` → `Check` (200ms flash) | `navigator.clipboard.writeText(secret)` | "Copy secret" |
| Generate | `Dices` | `generateSecret()` → sets secret state | "Generate random secret" |

Copy confirmation: icon swaps from `Clipboard` to `Check` for 1500ms then reverts. No toast needed — inline confirmation is sufficient.

Disabled states:
- Copy secret button: disabled when `secret === ''`
- All three buttons: fully interactive regardless of validity (user may want to generate a new secret to fix an error)

### Segmented Controls (ToggleGroup)

| Group | Options | Default |
|-------|---------|---------|
| Algorithm | `SHA-1` / `SHA-256` / `SHA-512` | `SHA-1` |
| Digits | `6` / `8` | `6` |
| Period | `30s` / `60s` | `30s` |

Guard: `onValueChange` must check `values.length > 0` before updating state — prevent deselection (see RESEARCH.md Pitfall 1).

ToggleGroup variant: `"outline"` (renders bordered pill-style segments).

### TOTP Code Display

| State | Visual |
|-------|--------|
| No secret (empty input) | Display `--- ---` (6-digit) in muted color (`text-muted-foreground`) |
| Invalid secret | Display `--- ---` (6-digit) or `---- ----` (8-digit) in muted color |
| Valid, generating | Display current code, e.g., `482 039` in `font-mono` 30px semibold |
| Code rotation | `animate-in fade-in duration-200` via `key={timeStep}` on code container |

Code format: `formatCode(code, digits)` — groups into halves with space. See RESEARCH.md Code Examples.

Copy button alongside code: `Clipboard` → `Check` flash (same pattern as secret copy).

### Countdown Progress Bar

| State | Visual |
|-------|--------|
| >66% remaining | Full-width bar, `bg-green-500`, transitions linearly |
| 33–66% remaining | Partial bar, `bg-yellow-500` |
| <33% remaining | Short bar, `bg-red-500` |
| Period boundary (reset) | `transition-none` for the reset tick to prevent slow 0→100 animation (see RESEARCH.md Pitfall 4) |
| No valid secret | Bar at 0%, `bg-muted`, no animation |

Bar height: 4px (`h-1`). Rounded ends: `rounded-full`. Wrapper background: `bg-muted` at 100% width with `rounded-full` to show the track.

Seconds label: right-aligned `text-sm text-muted-foreground` (`"12s"` format). Shown beside the bar.

## Animation Contract

| Animation | Trigger | Duration | Easing | Implementation |
|-----------|---------|----------|--------|---------------|
| TOTP code rotation fade | `key={timeStep}` changes | 200ms fade-in | `ease-out` | `animate-in fade-in duration-200` (tw-animate-css) |
| Progress bar width | Every 1s tick | 950ms linear | `linear` | CSS `transition: width 950ms linear` |
| Progress bar reset | Period boundary | `transition: none` | none | Remove transition class when `secondsRemaining === period` |
| Copy confirmation icon | Click copy button | 1500ms display | none | `useState` swap `Clipboard` → `Check`, `setTimeout` revert |
| Color change (green/yellow/red) | Threshold crossing | instant | none | Class swap on tick — no transition on color |

Note: Progress bar transition is 950ms (not 1000ms) to leave a 50ms gap before the next tick fires — prevents visual stutter at period boundary.

## Accessibility Notes

- Secret input: `aria-invalid={!!secretError}` when validation fails
- Error text: associated via proximity (adjacent sibling) — no explicit `aria-describedby` required at this fidelity, but planner may add it
- Icon buttons: each must have a unique `aria-label` (listed in Interaction States above)
- ToggleGroup: `@base-ui/react` provides ARIA role="group" and roving tabindex — no additional ARIA needed
- Progress bar: `role="progressbar"` with `aria-valuenow={secondsRemaining}` `aria-valuemin={0}` `aria-valuemax={period}`
- TOTP code: `aria-live="polite"` on the code container to announce rotation to screen readers
